Implementation Plan

Preview HTML — VS Code Extension

11 / 11 tasks complete

Overview

Build a VS Code extension that registers a Custom Editor as the default viewer for .html files. Users see rendered HTML on open (preview-first), with a toolbar button to toggle to the standard text editor. The preview live-updates on every document change with debouncing.

Architecture Decisions

Custom Text Editor API

Over Webview Panel — gives us default-editor registration, proper tab lifecycle, and native toggle support.

TypeScript + esbuild

Standard VS Code extension toolchain, fast bundling, minimal config.

No local server

Render HTML directly in the webview, rewrite asset paths with asWebviewUri().

Retain webview context

retainContextWhenHidden: true for instant tab switching without re-renders.

Small

Read the TextDocument content and render it inside the webview. The HTML file's content becomes the webview's HTML body.

Task 4: Resolve relative assets

Medium

Rewrite relative paths in the HTML to use webview.asWebviewUri() so they load correctly from disk.

  • Relative <img src="./image.png"> displays
  • Relative <link href="./style.css"> applies
  • Nested paths (../assets/img.png) resolve correctly
  • localResourceRoots set to file's parent + workspace folders

Task 5: Live update on document change

Small

Listen to onDidChangeTextDocument and re-render with a 300ms debounce.

  • Editing the file updates the preview within ~300ms
  • Rapid changes don't cause excessive re-renders
  • External file writes trigger an update
Checkpoint: Core Preview
  • Opening any .html file shows rendered preview by default
  • Relative assets load correctly
  • Editing the file externally updates the preview live
  • Extension builds and loads without errors

Risks & Mitigations

RiskImpactMitigation
Custom Editor API quirks with undo/redo Low We're read-only in the webview — no undo/redo needed.
Relative asset rewriting misses edge cases Medium Only rewrite clearly relative paths. Leave absolute URLs untouched.
Large HTML files cause slow re-renders Medium Debounce at 300ms. Consider incremental patching later.
Users confused by preview-first behavior Medium Clear README and a setting to disable.
Scripts in HTML pose security risk Low Webview sandboxed by VS Code. localResourceRoots restricts access.
